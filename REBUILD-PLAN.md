### Phase 2 — Rebuild source files from scratch (3–4 hrs)

Recreate from BUILD.md spec + my memory of how it worked. License format must match `license.dat` algorithm so existing customer keys still work.

- [ ] **`electron-src/main.src.js`** — Electron entry. Creates BrowserWindow only AFTER license validates. On invalid license, show activation.html. On valid, load ui/index.html. Disable DevTools in prod. Set CSP, disable nodeIntegration in renderer, contextIsolation:true.
- [ ] **`electron-src/preload.src.js`** — exposes `window.eORB.license`, `window.eORB.activate(key)`, `window.eORB.db.*` via contextBridge.
- [ ] **`electron-src/db.src.js`** — better-sqlite3 wrapper for the practice DB.
- [ ] **`electron-src/security/machine.src.js`** — fingerprint generator. **CRITICAL FIX:** must be deterministic across reboots. Use ONLY:
  - `node-machine-id` (REG.exe MachineGuid on Windows — stable)
  - OS hostname (stable)
  - Optional: CPU model from `os.cpus()[0].model` (stable on same hardware)
  - DO NOT include: free memory, current user list, network interfaces (MAC can change/disappear on Wi-Fi off), running processes, temp files, anything time-based.
  - Hash all three with SHA-256, return hex. Same machine = same hash forever (until OS reinstall).
- [ ] **`electron-src/security/crypto.src.js`** — AES-GCM encrypt/decrypt of license.dat using machine fingerprint as key (PBKDF2'd).
- [ ] **`electron-src/security/activation.src.js`** — `validateKey(key, customerId, email, masterSecret)`: HMAC-SHA256(`customerId|email.toLowerCase().trim()`, masterSecret), first 16 hex chars, uppercase, dashed every 4. Compare to user-entered key.
- [ ] **`electron-src/security/integrity.src.js`** — read `build/integrity-manifest.json`, SHA-256 each file at runtime, compare. On mismatch, refuse to start. SKIP this check during dev (`process.defaultApp`).
